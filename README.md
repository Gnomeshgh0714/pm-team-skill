# PM Team Skill — 多Agent产品经理团队 (v2.0)

## 这是什么

一套可复用的产品经理团队协作流程，封装为Claude Code / Hermes可用的Skill。支持完整模式（6阶段）和快速模式（4阶段），通过评审攻防机制确保方案质量。

## 快速开始

### 1. 安装

将 `pm-team-skill` 文件夹复制到 Claude Code 的 skills 目录：

```bash
# Claude Code skills 目录（选择你使用的路径之一）
cp -r ~/Desktop/pm-team-skill ~/.claude/skills/pm-team
# 或项目级
cp -r ~/Desktop/pm-team-skill /your-project/.claude/skills/pm-team
```

### 2. 使用

在 Claude Code 中直接对话：

```
请按照 pm-team skill 执行一轮完整的产品迭代。
产品方向：在线教育课程预约系统
P0功能：课程展示、预约管理、通知推送
P1功能：评价系统、课程推荐
P2禁令：不做支付、不做直播、不做社交
```

Claude Code 会自动：
1. 创建目录结构（team-rules/、research/、product/prd/、review/、growth/、retrospective/）
2. 按顺序执行6个阶段
3. 每阶段产出文件归档
4. 评审攻防 → 总监终审 → 修改 → 复审 → 定稿
5. 迭代结束后产出复盘报告

### 3. 单阶段使用

也可以只使用某个阶段：

```
请按照 pm-team skill 的阶段3，对以下PRD进行反向评审：
[粘贴PRD或提供文件路径]
```


## 6阶段工作流

```
① 用研输入 → ② 产品设计 → ③ 评审攻防 → ④ 总监终审 → ⑤ 落地执行 → ⑥ 复盘优化
```

| 阶段 | 执行者 | 交付物 | 关键规则 |
|------|--------|--------|---------|
| 用研输入 | 用户研究专员（子Agent） | 用研报告 | 痛点有场景，竞品≥3个 |
| 产品设计 | 产品设计专员（子Agent） | PRD+流程图+数据结构 | 提交前5项自检 |
| 评审攻防 | 产品评审专员（子Agent） | 评审意见 | ≥3条问题，只提问题不给方案 |
| 总监终审 | 产品总监（主Agent） | 终审决策+修改指令 | 逐项裁决，决策写入日志 |
| 落地执行 | 产品设计专员（子Agent） | 技术实施方案 | 任务拆解+时间节点 |
| 复盘优化 | 产品总监（主Agent） | 复盘报告 | ≥1优化点+≥1可沉淀技能 |

## 核心机制

### 评审攻防
评审专员必须提出实质性反对意见。实测第一轮迭代中发现5项严重问题（状态机死锁、数据模型错误、安全风险等），全部在开发前修复。

### 决策留痕
所有裁决写入决策日志，包含背景、内容、理由、生效时间。

### 能力升级
L1（走完流程）→ L2（发现问题）→ L3（沉淀技能）→ L4（适应新方向）

## Claude Code 适配说明

本skill利用Claude Code的子任务能力实现多角色协作：
- 主对话担任产品总监
- 子任务调度专业角色（用研/设计/评审/增长）
- 所有文件写入项目目录
- 不依赖任何外部工具或API

## 来源

基于「学术论文审稿协作平台」项目的完整实战经验提炼。该项目在单日内完成：
- 14个正式文件（约7000行）
- 10条决策记录
- 3个可沉淀技能
- 8个后端模块+21张数据库表
- L1+L2+L3验收通过
