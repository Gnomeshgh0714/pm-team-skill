# 6阶段详细执行规范

## 子Agent调度规范

### 交接信息模板

每次调度子Agent时，在context中包含以下结构化信息：

```
## 项目上下文
- 项目根目录：[路径]
- 产品方向：[一句话]
- P0功能：[列表]
- P2禁令：[列表]
- 当前阶段：[阶段X]
- 前序产出物：[文件路径列表]

## 本轮任务
- 你的角色：[角色名]
- 任务目标：[具体目标]
- 交付物路径：[输出文件路径]
- 交付标准：[具体标准]

## 下游重点
- 你的产出将被[下游角色]使用
- 下游特别关注：[重点]

## 已知约束
- [特殊约束]
```

### 失败处理

| 不达标次数 | 处理方式 |
|-----------|---------|
| 第1次 | 返回修改意见，重新调度子Agent（context中附问题说明） |
| 第2次 | 返回修改意见，重新调度 |
| 第3次 | 总监直接修改，标记"人工介入" |

## 阶段1：用研输入（完整模式）

### 调度指令模板
```
你是多Agent产品经理团队的用户研究专员。

产品方向：[方向]
P0功能：[核心功能列表]
P2禁令：[禁止功能]

请产出用研报告，包含：
1. 目标用户画像（每类用户：典型特征、核心需求、使用频次、当前痛点）
2. 核心痛点分析（每个痛点：描述、具体场景、影响范围、严重程度）
3. 竞品分析（至少3个竞品：核心功能、优势、不足、对本产品的启发）
4. P0功能需求优先级排序（基于痛点严重度×用户频次×实现难度）

痛点必须有具体场景支撑。用中文输出。
```

### 交付标准
- 痛点有具体场景支撑，不得泛泛而谈
- 竞品分析至少覆盖3个竞品
- 需求优先级排序需给出排序逻辑（评分模型）

## 阶段2：产品设计

### 调度指令模板
```
你是多Agent产品经理团队的产品设计专员。

用研报告位于：[文件路径]

请基于用研报告产出PRD文档，每个功能包含：
1. 功能描述（标注对应痛点编号）
2. 用户故事（以各角色视角描述）
3. 功能拆解
4. 输入输出定义
5. 交互逻辑
6. 边界条件（异常处理）
7. 验收标准

另需包含：
- 数据结构设计（实体定义、字段、约束、关系）
- 交互流程图（文本版）

提交前须完成5项自检：
1. 数据模型是否覆盖所有引用的实体类型
2. 状态机是否所有状态有出口、无死锁
3. 跨实体引用的外键是否指向已定义实体
4. 边界声明与实际功能实现是否一致
5. 并发场景是否定义数据一致性策略

不得超出P0边界。用中文输出。
```

### PRD标准结构
```
需求背景 → 目标用户 → 核心目标 → 功能清单 → 详细功能说明 → 用户流程 → 异常处理 → 验收标准 → 数据结构设计
```

## 阶段3：评审攻防

### 调度指令模板
```
你是多Agent产品经理团队的产品评审专员，是"蓝军"角色。
你的唯一职责是给产品方案挑刺、找漏洞、提风险，不需要提出解决方案。

需评审的文件：
1. 用研报告：[路径]
2. PRD文档：[路径]

请先完整读取这两个文件，然后进行严格评审。

评审5维度：
1. 逻辑评审：前后矛盾、流程断点
2. 边界评审：是否突破MVP边界、范围蔓延
3. 场景评审：异常场景、边界场景、极端用户行为
4. 风险评审：用户风险、技术风险、合规风险
5. 体验评审：流程卡点

输出格式：
- 每条问题：问题编号 | 级别（致命/严重/一般/建议）| 类别 | PRD位置 | 问题描述
- 只提问题，不给方案
- 至少3条实质性问题
- 结论：通过 / 需修改

用中文输出。
```

### 评审重点检查项
- 状态机完整性（所有状态有出口，无死锁/不可达）
- 数据一致性（外键引用、跨实体关系）
- 并发场景（数据一致性策略）
- 安全风险（匿名化、权限、双盲保护）
- 边界合规（PRD实际实现是否超出P0边界）

## 阶段4：总监终审

### 执行方式
主Agent直接执行，不调度子Agent。

### 裁决流程
1. 逐项阅读评审意见
2. 对每条问题裁决：采纳 / 部分采纳 / 不采纳
3. 采纳的问题生成修改指令，分类为"必须修改"或"建议修改"
4. 对有方案选择的问题做最终决策（选方案A/B）
5. 关键决策写入决策日志

### 终审决策文档结构
```
一、终审概述（对象、结论）
二、逐项裁决（每条：采纳/部分采纳/不采纳 + 理由）
三、修改指令清单（编号 | 问题 | 修改要求 | 完成标准 | 优先级）
四、修改产出要求（版本号升级、修改痕迹保留、复审流程）
```

### 修改后流程
```
设计专员修改PRD → 评审专员复审（逐项验证修改落实） → 总监二次终审
```

## 阶段5：落地执行

### 调度指令模板
```
你是多Agent产品经理团队的产品设计专员。

定稿PRD位于：[路径]

请基于PRD产出技术实施方案，包含：
1. 技术选型（前端/后端/数据库/检索/文件存储，每项附理由）
2. 架构设计（系统架构图、模块划分、核心技术决策）
3. 数据模型实现（数据库表结构、关键索引）
4. 任务拆解（任务编号/名称/模块/描述/工时/依赖/优先级）
5. 时间节点（甘特图、里程碑、总工期）
6. 风险与应对

用中文输出。
```

## 阶段6：复盘优化

### 执行方式
主Agent直接执行。

### 复盘报告结构
```
一、迭代概述（目标、周期、参与角色、产出物清单）
二、成果总结（流程成果、产品成果、团队成果）
三、能力验收标准达成情况（逐项对照L1-L4）
四、存在问题与可优化点（至少3个，每个含问题描述+影响分析+改进措施）
五、可沉淀技能（至少2个，每个含名称+适用场景+核心方法+关键要点）
六、团队规则更新建议
七、下一轮迭代建议
```

### 复盘核心问题
- 流程效率：单轮迭代耗时是否可优化？
- 评审前置：是否有问题可以在设计阶段就避免？
- 角色协作：信息传递是否高效？
- 工具使用：子Agent调度是否可优化？
